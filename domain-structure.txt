## Proposed File Structure for Schema Splitting

```
prisma/
├── schema.prisma (main file with generator/datasource)
├── domains/
│   ├── abilities.schema.prisma
│   ├── contests.schema.prisma
│   ├── encounters.schema.prisma
│   ├── evolution.schema.prisma
│   ├── generations.schema.prisma
│   ├── items.schema.prisma
│   ├── languages.schema.prisma
│   ├── machines.schema.prisma
│   ├── moves.schema.prisma
│   ├── natures.schema.prisma
│   ├── pal-park.schema.prisma
│   ├── pokemon.schema.prisma
│   ├── pokemon-characteristics.schema.prisma
│   ├── regions.schema.prisma
│   ├── stats.schema.prisma
│   └── types.schema.prisma
```

## Notes:
- Each domain file will contain only the models listed under that domain
- Cross-domain relationships will be maintained through foreign key references
- The main schema.prisma will import all domain files
- Sub-domains are logical groupings within each domain file
